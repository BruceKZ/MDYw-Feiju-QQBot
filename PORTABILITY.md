# 项目可移植性与迁移报告 (Portability & Migration Guide)

基于项目的架构（Docker 容器化编排），该项目的**整体跨平台可移植性极佳**。在 Windows、Mac 或 Linux 系统环境之间互相切换开发或部署时，只需确保具备 Docker 基础环境即可，业务逻辑层级目前没有对特定操作系统的强耦合依赖。

以下是关于如何安全地将项目迁移到新设备的详细指南及注意事项：

## 1. 基础环境级迁移 (Environment)
*   **依赖说明**：项目高度依赖 Docker 和 Docker Compose。更换到新设备时，只需提前按照系统类型安装好 [Docker Desktop](https://www.docker.com/products/docker-desktop/)（Windows/Mac 环境）或 Docker Engine（Linux 环境）。
*   **代码迁移**：直接使用 `git clone` ，或者拷贝整个代码库文件夹即可。
*   **一键恢复**：大部分 Python 依赖和底层处理环境都被密封在自带的 `Dockerfile` 中。由于环境隔离，新电脑**不需要在宿主机重新配置 Python 虚拟环境**，直接在项目根目录拉起终端执行 `docker-compose up -d --build` 即可一键恢复后端运行。

## 2. 核心数据迁移指南 (Data Migration)
由于涉及个人隐私凭证、动态 Session 数据以及体积庞大的 SQLite 数据库文件均在 `.gitignore` 中被排除同步，**单纯依靠 Git 拉取代码是无法同步业务历史状态的**。
如果您想在新电脑上实现“**完全免扫码登录复原**”并继承所有的配置状态，**必须通过 U盘/网盘 等方式手动拷贝转移**以下被忽略的核心路径：

**需要打包转移的关键清单：**
1.  **全局私密变量 (`.env`)**：
    *   位于根目录的 `.env` 文件。包含了服务间鉴权的 Token 以及分配的超级管理员 QQ。
2.  **NapCat 的框架及 QQ 登录态数据**：
    *   `napcat/config/` (包含框架层级相关的登录行为属性)。
    *   `napcat/qq/` (⚠️ 此为极度关键目录：包含了您的账号设备锁 Session、缓存和私聊信令加密密钥。如果不迁移它，在新设备启动后会被腾讯视为“新设备”，将强制要求重新扫码登录验证)。
3.  **NoneBot 原生数据库**：
    *   位于 `mdy_feiju/data/` 目录下的所有文件。必须随项目一同拷贝，主要包含：
        *   `2fa.db` 与 `2fa.key`：双重要求认证插件的数据库和独立主加密密钥库。**注意：这非常关键，一旦漏拷，新电脑生成的 2FA 数据将解密失败。**
        *   `memes.db`：管理您所拥有的全部“自定义梗图/表情包”映射记录。
        *   `shared_db/` 以及其他所有留存在这里的 SQLite3 数据表和多媒体缓存。

**💡 最佳实践**：在设备替换时，最省心、零错误率的方案是：**避开 Git，全盘选中整个 `MDYw-Feiju-QQBot` 文件夹打包成 `.zip`，解压到新环境后直接 `docker-compose up -d` 启动**。

## 3. 换电脑时可能遇见的外部问题 (Potential Issues)
在新的物理机上启动时，请注意排查宿主机特征冲突：
*   **端口被宿主机占用冲突**：
    *   在确保新设备未打开占网应用的情况下执行容器化启动。项目目前占用的对内/对外物理映射端口有：`6099` (Napcat WebUI)、`3001` (Napcat WebSocket) 和 `8080` (Nonebot Webhook API)。
*   **平台耦合风险说明**：
    *   经过对工程代码的重构盘点，曾携带有 Windows 硬编码全路径字体(`C:\Windows\Fonts\msyh.ttc`)依赖的异常组件(`wordcloud` 词云功能) 已被物理销毁。项目目前**已达到 100% 服务端跨平台无差别化就绪状态**，不会由于字形或平台渲染层面的隐患产生报错。
